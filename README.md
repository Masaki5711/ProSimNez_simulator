# ProSimNez - 生産ネットワークシミュレーター

Witness風の離散イベントシミュレーションエンジンを搭載した、工場の生産フローをビジュアルに設計・シミュレーション・分析できるWebアプリケーションです。

## スクリーンショット

**シミュレーション実行画面**
- ライブフロービュー（工程カード＋SVG接続線＋リアルタイム更新）
- 工程テーブル（稼働率バー＋ボトルネック強調）
- 搬送ライブモニター（全ルート常時表示）
- タイムライン分析（ガントチャート＋在庫推移＋生産累計）

## 主な機能

### ネットワーク設計
- **ドラッグ＆ドロップ**でプロセスノード（加工・組立・検査）とストアノードを配置
- **接続線**で工程間の搬送ルートを定義（AGV / コンベア / 手動 / フォークリフト）
- **BOM（部品表）管理** — 製品の親子関係と必要数量を定義
- **材料設定** — 各工程の投入材料・出力製品・バッファ設定・スケジューリング方式

### シミュレーション
- **離散イベントエンジン** — SimPyベースの非同期シミュレーション
- **BOM連動** — BOM定義に基づく材料消費（素材A×1 + 素材B×1 → 加工品×1）
- **バッファ容量制限** — INバッファ満杯→搬送停止、OUTバッファ満杯→工程ブロック
- **Push/Pull/Kanban** — スケジューリング方式の切替（カンバン枚数制限対応）
- **不良率シミュレーション** — 工程ごとの不良率設定
- **搬送ロットサイズ** — 製品別ロットサイズとキャパシティ制限
- **ストア制御** — 部品ストア（プル補充）/ 完成品ストア（容量制限）
- **稼働時間設定** — 完成品ストアの稼働時間からシミュレーション時間を自動計算

### リアルタイム監視
- **WebSocket**経由のライブデータ配信
- **ライブフロー** — 工程カード＋稼働率バー＋搬送アニメーション
- **KPIダッシュボード** — 完成品数・スループット・稼働率・品質率・ボトルネック
- **搬送モニター** — 全搬送ルートの稼働状況を常時表示

### 分析・レポート
- **タイムライン分析** — 工程ガントチャート（稼働/待機/ブロック）
- **在庫推移グラフ** — バッファ別の在庫変動
- **生産累計カーブ** — 工程別の生産推移
- **PDF出力** — シミュレーション条件＋結果の詳細レポート
- **CSV出力** — 工程別・搬送別データのエクスポート

- Python 3.10+

```bash

pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

```bash
cd frontend
npm install
npm start
```

## 使い方

### 1. プロジェクト選択
- サイドバー「プロジェクト」からプロジェクトを選択
- 初回起動時にデモプロジェクトが自動作成される

### 2. ネットワーク設計
- 「ネットワーク設計」タブでノードを配置
- 工程ノードをダブルクリック → 基本設定（CT・設備数・バッファ容量）
- 工程ノードを右クリック → 「材料設定」で投入材料・出力製品を定義
- 接続線を右クリック → 搬送方式・ロットサイズを設定
- ストアノードをダブルクリック → 容量・稼働時間・生産スケジュール

### 3. 部品編集
- 「部品編集」タブで製品マスタとBOM構成を管理

### 4. シミュレーション実行
- 「シミュレーション」タブで「開始」ボタン
- 速度切替: 10x / 60x / 600x / MAX
- ライブフロー＋テーブルでリアルタイム監視
- 完了後「タイムライン分析」タブでガントチャート表示
- 「PDF出力」ボタンで詳細レポート生成

### 5. サイドバー折りたたみ
- サイドバーの「←」ボタンまたは Ctrl+B で非表示
- シミュレーション画面を全幅で表示可能

## シミュレーションエンジン仕様

### 離散イベントシミュレーション
- SimPyベースの非同期エンジン
- 時間加速（1秒～10000倍速）
- 1秒ステップで工程進行・搬送・補充を処理

### 材料フロー
1. **ストア** → AGV/CNV搬送 → **工程INバッファ**
2. **工程** — BOMに基づき材料消費 → 加工（CT秒） → 出力
3. **工程OUTバッファ** → 搬送 → 次工程INバッファ
4. **最終工程** → 搬送 → **完成品ストア**

### ブロック機構
- OUTバッファ満杯 → 工程ブロック（加工停止）
- INバッファ満杯 → 搬送停止（AGV待機）
- 完成品ストア満杯 → ライン全体がカスケードブロック

### スケジューリング
| 方式 | 動作 |
|---|---|
| Push | バッファに空きがあれば常時搬送 |
| Pull | 下流INが安全在庫×2以下で搬送 |
| Kanban | カンバン枚数で同時搬送件数を制限 |
